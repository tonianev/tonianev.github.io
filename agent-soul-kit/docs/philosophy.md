# Philosophy

Agent Soul Kit treats character as infrastructure.

That means:

- The contract is legible.
- The files are versioned.
- Behavior changes are diffable.
- The agent is not allowed to fake capabilities.
- The point is better work, not cosplay.

## Why three files

`soul.md` captures how the agent should feel.

`memory.md` captures what it should remember.

`posture.md` captures how it should move when the next step is not obvious.

Keeping those separate prevents one giant prompt from becoming unmaintainable.

## What success looks like

- The agent is recognizable across tasks.
- It keeps durable context without overstating memory.
- It asks fewer low-value questions.
- It feels more human while staying operationally honest.
